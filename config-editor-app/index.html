Focus settings

If this toggle is in the on position, then audio notifications will not play while any "Focus" is turned on. Visual notifications will still be sent (though they may be suppressed by macOS unless you've allowed terminal-notifier in your Focus settings).

How does this solution work from a technical standpoint?

Our tool integrates with Claude Code's hooks system, which allows external scripts to execute when specific events occur during Claude Code sessions.

Installation process
When you open this application for the first time, it automatically installs notification scripts to ~/.claude/scripts/ and configures Claude Code's ~/.claude/settings.json file to execute these scripts when hook events fire. No manual configuration is required.

Supported hook events

  • Notification - Runs when Claude Code sends notifications.
  • Stop - Runs when Claude Code finishes responding.
  • PreToolUse - Runs before tool calls (permission prompts).
  • PostToolUse - Runs after tool calls complete.
  • SubagentStop - Runs when subagent tasks complete.

Audio delivery
Audio notifications are played using macOS's native afplay command, which plays audio files directly without requiring any background processes. Sound files are stored in ~/.claude/voices/ and can be customized per hook type.

Visual notifications
Visual notifications are delivered via terminal-notifier, which sends notifications to macOS Notification Center. These notifications appear alongside your other system notifications and persist in notification history. By default, terminal-notifier notifications are silent unless you've enabled sounds for it in System Settings. When Focus mode is active, these visual notifications may be suppressed by macOS unless you've added terminal-notifier to your Focus mode's allowed apps list.

Focus mode detection
When Focus mode control is enabled in settings, the notification scripts check macOS's current Focus status using native macOS commands before playing audio notifications. This allows audio to respect Do Not Disturb, Sleep, Work, and other Focus modes without requiring additional setup. Visual notifications are always sent regardless of this setting, though macOS may suppress them based on your Focus configuration.

Configuration storage
All settings (sound preferences, enabled/disabled hooks, Focus mode preferences) are stored locally in ~/.claude/config.json. Your analytics preference is stored in browser localStorage. The GUI application reads and writes to these files, providing a visual interface for configuration that would otherwise require manual JSON editing.

No background process
Unlike many notification solutions, our tool does not require a background process to remain running. Once installed and configured, the notification scripts are executed directly by Claude Code's hooks system when events occur. You can close the GUI application immediately after installation.

Do Apple "Focuses" (e.g. "Do Not Disturb," Work, Sleep) silence this app's notifications?

Visual notifications
By default, visual notifications are suppressed by Focus modes. However, you can configure your Focus settings to allow notifications from terminal-notifier to break through. Visual notifications use macOS Notification Center (via terminal-notifier). When Focus mode is active, macOS suppresses these notifications unless you've specifically allowed terminal-notifier in your Focus settings. Learn how to allow visual notifications during Focus modes.

Audio notifications
Configurable. By default, audio plays regardless of Focus mode to ensure you never miss important Claude Code events. Audio plays directly via afplay, which bypasses Notification Center, giving you control over whether it respects Focus modes.

How to silence audio during Focus modes
Enable the "Silence this application 'when you have Focus turned on'" toggle in the Focus settings section. This works with all macOS Focus modes including Do Not Disturb, Sleep, Work, Personal, and custom Focus modes. Detection happens automatically using built-in macOS commands - no additional setup required.

How do I allow visual notifications during Focus modes?

By default, macOS suppresses visual notifications from terminal-notifier when any Focus mode is active. You can configure your Focus settings to allow these notifications through:

  1. Open System Settings → Focus
  2. Select the Focus mode you want to configure (e.g., "Do Not Disturb" or "Work")
  3. Scroll to the Apps section
  4. Click the + button
  5. Search for and add terminal-notifier

Note: terminal-notifier may not appear in the app list until you've received at least one notification from it. Try triggering a notification first, then return to Focus settings to add it.

Use case: This is particularly useful if you're frequently on calls and need silent notifications. Disable audio notifications via the Claude Code tab, add terminal-notifier to your Focus allowed apps, and you'll receive silent visual banners without any audio interrupting your meetings.

How do I uninstall this application?

Step 1: Remove relevant values from ~/.claude/scripts/ and ~/.claude/settings.json

Click the button below to remove notifications-related values from ~/.claude/scripts/ and ~/.claude/settings.json. Please note that all active Claude Code sessions will need to be restarted if these notifications values are removed.

An error message will display if after these files are modified a hook event fires before a previously active Claude Code session has been restarted.

Note: You'll need to start a new Claude Code session for the changes to take effect. Your current session will continue showing hook errors until you restart.

If a Claude Code event fires before you restart, you may see an error like: Stop [bash /Users/.../.claude/scripts/smart-notify.sh stop] failed with non-blocking status code 127: No such file or directory. This is expected and harmless—it simply means Claude Code is trying to call the notification script that was just removed.

Step 2: Uninstall the desktop app
After removing the notification hooks above, you can delete this desktop application by moving it to the Trash from your Applications folder.

What data does this app collect?

Anonymous usage metrics
By default this app collects anonymous usage metrics to help us understand how people use it and improve it over time. You can opt out at any time via switching off the "Help improve this app by sharing anonymous usage data" toggle.

What we collect

  • Events like "app opened," "hook toggled," "sound changed," "settings saved."
  • Whether specific features are enabled or disabled (e.g., Focus mode control).
  • Basic platform information (macOS version, app version).
  • Error messages and stack traces (limited to 500 characters) when crashes occur.

We don't collect

  • Personal information (name, email, IP address).
  • File paths or project names.
  • Code or file contents.
  • Session recordings or screenshots.
  • Precise location data.
