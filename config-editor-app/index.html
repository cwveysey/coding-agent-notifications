Focus settings

If this toggle is in the on position, then this application's audio will not play while any "Focus" is "turned on." If this toggle is in the off position, then this application's audio will play regardless of whether or not a Focus is turned on.

Do Not Disturb, Work, and Sleep are all examples of what Apple refers to as "Focuses."

Some customization details

Using the controls above:

  • You can turn audio alerts on and off on a per hook type basis.
  • You can arrange for different audio to play on a per hook type basis.
  • You can arrange for custom audio to be used for Claude Code audio alerts.
  • You can choose whether or not Claude Code audio alerts are silenced if any Apple "Focuses" are turned on.

How does this solution work from a technical standpoint?

Our tool integrates with Claude Code's hooks system, which allows external scripts to execute when specific events occur during Claude Code sessions.

Installation process
When you open this application for the first time, it automatically installs notification scripts to ~/.claude/scripts/ and configures Claude Code's ~/.claude/settings.json file to execute these scripts when hook events fire. No manual configuration is required.

Supported hook events

  • Notification
    Runs when Claude Code sends notifications.
  • Stop
    Runs when Claude Code finishes responding.
  • PreToolUse
    Runs before tool calls (permission prompts).
  • PostToolUse
    Runs after tool calls complete.
  • SubagentStop
    Runs when subagent tasks complete.

Audio delivery
Audio notifications are played using macOS's native afplay command, which plays audio files directly without requiring any background processes. Sound files are stored in ~/.claude/voices/ and can be customized per hook type.

Visual notifications
Visual notifications are delivered via terminal-notifier, which sends notifications to macOS Notification Center. These notifications appear alongside your other system notifications and persist in notification history.

Focus mode detection
When Focus mode control is enabled, the notification scripts check macOS's current Focus status using native macOS commands before playing audio. This allows audio to respect Do Not Disturb, Sleep, Work, and other Focus modes without requiring additional setup.

Configuration storage
All settings (sound preferences, enabled/disabled hooks, Focus mode preferences) are stored locally in ~/.claude/config.json. The GUI application reads and writes to this file, providing a visual interface for configuration that would otherwise require manual JSON editing.

No background process
Unlike many notification solutions, our tool does not require a background process to remain running. Once installed and configured, the notification scripts are executed directly by Claude Code's hooks system when events occur. You can close the GUI application immediately after installation.

What third-party tools does this app use?
  • terminal-notifier
    macOS command-line tool for delivering user notifications to the macOS Notification Center.
  • Claude Code
    The hooks system that enables these notifications to trigger at the right moments.
  • Tauri
    Framework used to build this desktop application.

Do Apple "Focuses" (e.g. "Do Not Disturb," Work, Sleep) silence this app's notifications?

Visual notifications
Always blocked by Do Not Disturb and Focus modes - this cannot be changed. Visual notifications use macOS Notification Center (via terminal-notifier), which is controlled by the operating system. When DND/Focus mode is active, macOS blocks all visual notifications automatically.

Audio notifications
Configurable. By default, audio plays regardless of Focus mode to ensure you never miss important Claude Code events. Audio plays directly via afplay, which bypasses Notification Center, giving you control over whether it respects DND.

How to silence audio during Focus modes
Enable the "Silence this application 'when you have Focus turned on'" toggle in the Focus settings section. This works with all macOS Focus modes including Do Not Disturb, Sleep, Work, Personal, and custom Focus modes. Detection happens automatically using built-in macOS commands - no additional setup required.
